Register

These settings will be applied to all the devices using the desktop or app to scan QR codes, so keep in mind that any changes made by any user with the permissions to edit these settings will be reflected in all devices.

Automatically mark the scanned tickets or add-ons as registered

When this option is enabled, tickets will be marked as "checked-in" or an add-on "redeemed" after the QR code is scanned and is validated. If this option is disabled, you will need to confirm ticket check-in or add-on redeem on the popup window. See example.

Check attendee's ID at the event entrance

Ask for the attendee's identification for verification, if it is required to grant access to the event (e.g., to check the full name or age).

Check order confirmation

Ask for the order confirmation sent to the attendee's email, in case the QR scan does not work and the registration is done through the search box, as an additional security measure.

Re-entry allowed

If enabled, scanned tickets can be used multiple times to re-enter the event, useful for events with multiple sessions or breaks. If disabled, once a ticket is scanned and checked in, it cannot be reused to re-enter the event.

Custom check-in message

Display a custom message after checking in the attendee, to notify the staff of a special ticket type checked-in, for special handling or access to specific areas. Can add up to 3 messages. See example.

    Show table data

    Show all the event orders data in the table. It is recommended to disable this option, as showing the table can cause the staff to check-in wrong attendees or redeem wrong add-ons or expose orders data to other attendees. If disabled, only the tickets or orders matching the search criteria will be shown.

    Create a generic username and access code:

    Create a generic username and password to be shared with staff for scanning or manually validating pre-defined tickets and/or add-ons.

    It is recommended that each staff that will scan QR codes have its own Axesum account and that you give them permission to scan QR codes in the "Roles" and "Permission" pages, for increased security and control. But for simplicity, you can use the same username and password for all, by sharing the same access credentials with them. Can add up to 5 generic roles.
